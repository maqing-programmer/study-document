1       分支管理

1.1. 主分支

主分支即master分支，用来部署生产环境的分支。

1.2. 开发分支

在 git 中，每个分支都是独立的项目，有着各自的版本管理。

代码在开发过程中，根据需求创建的分支称为开发分支。

开发分支的创建基于当前的主分支。

开发分支是并发的、独立互不影响。

一旦完成开发，开发分支单独进测，测试通过，合并到主分支。

1.3. 分支命名

分支名唯一，不重复。

开发分支名使用story + $story_id 的方法命名，例如story6335。

如果一个分支含有多个story，那么使用task + $task_id 命名。

2       获取代码

1.  

2.  

2.1. 本地安装git，配置git

Windows用户从http://www.git-scm.com/download/下载安装

打开git bash，使用命令

git config --global user.name "yourName"

git config --global user.email "yourName@themakers.cn"

注意替换yourName

2.2. 本地生成SSH秘钥

打开git bash，

使用命令ssh-keygen -t rsa -C "yourName@themakers.cn"，一路回车即可。

2.3. 配置你的github账户

注册并登陆github

访问https://github.com/settings/keys，点击New SSH key，添加SSH key。

用记事本打开你本地的id_rsa.pub文件，

Windows用户该文件在"%HOMEPATH%/.ssh/"路径下

将id_rsa.pub文件的全部内容复制粘贴到Key的内容区域，点击保存。

2.4. 从github仓库克隆代码

打开git bash，使用命令

git clone git@github.com:ufzt/themakers.git

从仓库克隆整个项目的代码

3       初次提交

3.  

3.1. git的提交

在git里，代码提交一不需要更新代码二不需要与服务器通信。

简单理解，本地随意提交代码。

3.2. git add

使用命令 git status 查看本地的修改情况

如果你没有修改过文件，你什么也看不到

如果你修改过文件a，你会看到文件a会被列出，并且会显示成红色

 

git的提交内容需要你主动add，与你修改了几个文件没有关系。

比如你修改了a和b两个文件，你只有add了a和b两个文件，才会提交这两个文件。

 

你可以通过绝对路径 add 文件，例如使用命令

git add module/eventsMod.class.php

来add eventsMod.class.php文件

 

使用git status查看，add后的文件会单独列出来，并且会显示成绿色。它们会在你提交时被提交。

3.3. git commit

提交之前，你需要 git add所有需要提交的文件

使用git status 确认你所有要提交的文件

使用git diff --cached 查看你即将要提交的所有改动

最后，使用命令git commit -m '你提交了什么' 提交你的代码

你提交了什么 是你提交的备注，一行以内，言简意赅

3.4. git reset

git add的可逆操作，git reset

在提交前，你忽然不想提交b文件了，但之前你已经add了a和b两个文件

使用命令 git reset module/eventsMod.class.php 来取消add 文件

使用git status查看，绿色标记里没有了b文件，红色标记里出现了b文件

3.5. git checkout

如果你不小心删除了整个api文件夹，你不需要去仓库拉代码。

使用命令git checkout api，你会发现删除的文件夹恢复了。Git保留的是全部的代码。

 

如果你修改了b文件，你也不打算提交它，你想撤销对b文件的修改。

使用命令git checkout module/eventsMod.class.php

文件回到了修改前的状态

3.6. git push

你的功能开发完了，你可以把你的分支推到仓库。

只有开发分支可以推送到仓库，主分支(master分支)，不允许推到仓库。

使用命令git branch -m story6335 可以重命名当前分支

使用命令git push origin story6335 将你的分支推送到仓库

4       迭代提交

4.  

4.1. 新的开发还是迭代开发？

你的开发是否基于之前的分支？（选项A）

或者你需要从主分支开始全新的开发？（选项B）

4.2. 本地分支

你的开发是基于之前的分支？

git branch命令用于查看本地分支，标有*的分支是当前分支

你可以通过命令

git fetch origin 仓库分支名:本地自拟名

把仓库分支拉回本地

再通过命令 git checkout 本地分支名，切换到本地目标分支

4.3. 新的开发

你需要从主分支开始全新的开发？

用命令git fetch origin master:story4567, 创建基于主分支的开发分支story4567。

git checkout story4567，切换到开发分支story4567

 

你可以按照git add, git commit步骤，逐步提交你的代码。

记住每天git push你的分支到仓库。
